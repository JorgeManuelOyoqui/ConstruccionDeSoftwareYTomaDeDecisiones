¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es una inmensa red de computadoras alrededor de todo el mundo conectadas entre sí. En cambio, la web (la World Wide Web) es una enorme colección de páginas que se asienta sobre esa red de computadoras. (BBC, 2019)

¿Cuáles son las partes de un URL?

Las partes de una URL son:

  • Esquema: Son los protocolos de comunicación en red entre los servidores web y los navegadores. Transmiten mensajes, recuperan información a través del sistema de nombres de dominio (DNS) y la devuelven al navegador. (La diferencia entre HTTP y HTTPS es que este último cifra el proceso de transmisión de datos.) (Hostinger, 2025)
  • Subdominio: Indica que un sitio web es accesible a través de Internet y utiliza HTTP para comunicarse. El www (World Wide Web) es el más común. (Hostinger, 2025)
  • Nombre del dominio: es lo que los usuarios escriben en la barra de direcciones de su navegador para llegar a un sitio web. (Hostinger, 2025)
  • Extensión de dominio: También llamado dominio de nivel superior (TLD), es el bit que sigue al nombre de un sitio web. Algunos ejemplos son: .com, .mx, .org, .net, etc. (Hostinger, 2025)
  • Ruta: Una ruta al recurso proporciona información adicional a un servidor web, permitiéndole llevar a los usuarios a una ubicación específica. (Hostinger, 2025)
  • Parámetros: Son cadenas de consulta o variables de la URL. Son la parte de una URL que sigue a un signo de interrogación. Los parámetros más comunes son los de traducción, búsqueda, filtrado o seguimiento. (Hostinger, 2025)
Hostinger (2025). “¿Qué es una URL) Ejemplos, estructura y tipos”. Recuperado de: https://www.hostinger.com/mx/tutoriales/que-es-una-url

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET: Solicita una representación de un recurso específico (Por ejemplo, HTML, JSON o JavaScript). (MDN Web Docs, S.F.)
  • HEAD: Pide uns representación parecida al de GET, sólo que sin el cuerpo de la respuesta, pues a este método no le interesa el contenido sino que se interesa sólo por el Header (de ahí su nombre). (Williams Morales, S.F)
  • POST: Envía una entidad a un recurso en específico, causando un cambio en el estado o efectos secundarios en el servidor web. (MDN Web Docs, S.F.)
  • PUT: Reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición. (MDN Web Docs, S.F.)
  • PATCH: Se usa para aplicar modificaciones parciales a un recurso. (MDN Web Docs, S.F.)
  • DELETE: Borra un recurso en específico. (MDN Web Docs, S.F.)
MDN Web Docs (S.F.). “Métodos de petición HTTP”. Recuperado de: https://developer.mozilla.org/es/docs/Web/HTTP/Reference/Methods

Williams Morales (S.F.). “Métodos HTTP - POST, GET, PUT, DELETE”. Recuperado de: https://estilow3b.com/metodos-http-post-get-put-delete/

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Se debe utilizar POST, ya que es un método que sirve para enviar datos al servidor de manera segura y permitir que se procesen o almacenen, como en el caso de una autenticación.

Por lo que, a diferencia de GET que incluye los datos en la URL, POST envía los datos al cuerpo de la solicitud HTTP, lo que significa que no quedan visibles en la barra de direcciones del navegador, lo que aumenta la seguridad de la información sensible como contraseñas.

MDN Web Docs (S.F.). “Post”. Recuperado de: https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Methods/POST

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Se usa GET, pues se usa para solicitar datos desde un servidor sin modificar ningún recurso en el servidor. Aparte, también es el que se utiliza para recuperar información sin enviar datos al servidor. Por eso es que se usa GET.

MDN Web Docs (S.F.). “GET”. Recuperado de: https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Methods/GET

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

No, todo lo contrario. Cuando la respuesta HTTP es de un código 200 significa OK, lo que indica que la petición fue exitosa y que no ocurrió ningún error.

Williams Morales (S.F.). “Métodos HTTP - POST, GET, PUT, DELETE”. Recuperado de: https://estilow3b.com/metodos-http-post-get-put-delete/

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

El error 404 indica que el servidor NO pudo encontrar la página o recurso solicitado. Por ende, sí es responsabilidad del desarrollador corregir este error, ya que se debe a errores de mantenimiento, en la optimización de los motores de búsqueda (como google) o en la gestión o estructura del sitio; y al ser errores internos y no externos del usuario, entonces deben ser corregidos por el o los desarrolladores.

Williams Morales (S.F.). “Métodos HTTP - POST, GET, PUT, DELETE”. Recuperado de: https://estilow3b.com/metodos-http-post-get-put-delete/

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

El error 500 es un error interno del servidor, y hay de muchos tipos. Al ser un error interno relacionado con problemas técnicos, el mantenimiento de las funcionalidades del sitio o debido a la falta de prevención de recurrencias entonces sí es responsabilidad del o los desarrolladores solucionar ese problema.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Si un atributo HTML5 es despreciado o desaprobado es porque su uso ya no es recomendado debido a que hay alternativas mejores y más eficientes.

Algunos ejemplos de atributos HTML 4 que estén desaprobados en HTML5 son:

  • acronym: para definir acrónimos y reemplazado por abbr en HTML5.
  • applet: para incrustar miniaplicaciones Java. Se reemplazó por "object" o "embed".
  • basefont: para definir la fuente base para el texto de una página. Se reemplazó con el uso de CSS para el manejo de estilo de fuentes.
  • big: para aumentar el tamaño de fuente del texto. Al igual que con basefont, se reemplazó con el uso de CSS para el manejo de estilo de fuentes.
  • center: para centrar el contenido. Igualmente, se reemplazó con el uso CSS para alinear el contenido.
  • tt: para mostrar el texto de una fuente monoespaciada. Se reemplazó con el uso de CSS con la propiedad font-family.
  • u: para subrayar el texto. Se reemplazó con el uso de CSS con la propiedad text-decoration: underline;.

Siendo estos sólo algunos ejemplos de todos los que hay.

MDN Web Docs (2024). “HTML: Lenguaje de etiquetas de hipertexto”. Recuperado de: https://developer.mozilla.org/es/docs/Web/HTML

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML5 es la versión mejorada de HTML 4, incluyendo mejoras y nuevas funcionalidades para adaptarse a las necesidades del desarrollo web, como:

  • Elementos semánticos con nuevas etiquetas para una estructura más clara como con las etiquetas "header", "footer", "article", "section", "nav", etc.
  • Mejor soporte de multimedia permitiendo introducir las etiquetas de "audio" y "video", permitiendo la reproducción de este tipo de archivos.
  • Nuevos tipos de controles de formularios como email, url, number, range, date, color, entre otros; facilitando la validación.
  • HTML5 posee almacenamiento local de modo que se tiene una mayor capacidad y control sobre los datos que se almacenen localmente.
  • Incorpora el elemento "canvas" para el uso de animaciones y gráficos usando JavaScript.
  • Incluye Geolocalización de modo que se puede conocer la ubicación geográfica del usuario.
  • Posee mejor compatibilidad con navegadores modernos que HTML 4, mejorando así el rendimiento y capacidades del HTML5.

Estos son sólo algunos ejemplos de las diferencias que hay entre el HTML 4 y el HTML5, donde se puede ver que el HTML5 es una mucho mejor versión que el HTML 4, siendo más moderno, con mejor rendimiento y con mayores capacidades.

¿Qué componentes de estructura y estilo tiene una tabla?

Algunos de los principales elementos que conforman la estructura y estilo de una tabla son:

Para la estructura:

  • table: Elemento que define la tabla.
  • caption: Sirve para darle título o descripción a la tabla.
  • thead: Agrupa las filas que contienen los encabezados de las columnas.
  • tbody: Agrupa las filas de la tabla que contienen los datos principales de la tabla.
  • tfoot: Agrupa las filas que contienen la información del resumen o pie de la tabla, como los totales por ejemplo.
  • tr: Para definir una fila de la tabla.
  • th: Para definir una celda de encabezado de una fila. Normalmente se usa dentro de thead.
  • td: Para definir una celda de datos en una fila.. Se usa dentro del tbody.

Para el estilo, aunque se suele usar CSS para esto pero HTML5 igual permite aplicar atributos de estilo gracias a style, lo que incluye:

  • Bordes: Para definir los bordes de la tabla y sus celdas por medio de la propiedad border.
  • Espaciado: donde se usa padding para controlar el espaciado interno de una celda y se usa border-spacing para manejar el espacio entre celdas.
  • Color de fondo: donde se pueden usar selectores como :nth-child(even) para cambiar los colores de fondo en las filas.
  • Alinear texto: donde se usan las propiedades text-align y vertical-align para controlar la alineación del contenido de una celda.

¿Cuáles son los principales controles de una forma HTML5?

Algunos ejemplos de los principales controles para formularios en HTML 5 son:

  • input type=“text”: Para la entrada de una sola línea de texto.
  • textarea: Para la entrada de varias líneas de texto.
  • input type=“checkbox”: Permite seleccionar una o varias opciones de un conjunto de opciones.
  • input type =“radio”: Para facilitar el seleccionar una opción de varias otras disponibles.
  • select: Muestra una lista de opciones de un menú desplegable.
  • input type =“submit”: Para enviar el formulario para que sea procesado.
  • input type =“reset”: Para restablecer todos los campos del formulario a sus valores iniciales.
  • input type =“email”: Para validar que la entrada sea una dirección válida de correo electrónico.
  • input type =“url”: Para validar que la entrada sea una url válida.
  • input type =“number”: Para ingresar solo valores numéricos, incluyendo opciones para definir mínimos y máximos.
  • input type =“range”: Da un control deslizante para seleccionar un valor de un rango definido.
  • input type =“date”, input type =“time”, input type =“datetime-local”, input type =“month”, input type =“week”: Para facilitar la selección de fechas u horas en distintos formatos.
  • input type =“search”: Permite la entrada de términos de búsqueda.
  • input type =“tel”: Permite la entrada de números telefónicos válidos.
  • input type =“color”: Proporciona una interfaz para seleccionar colores.

Sobre el ciclo de vida y desarrollo de los sistemas de información: ¿Cuál es el ciclo de vida de los sistemas de información? ¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de vida de los sistemas informáticos (SDLC) describe las etapas por las que pasa un sistema desde la concepción hasta el momento que sea retirado. El ciclo de desarrollo de sistemas de información es un proceso que guía el desarrollo de sistemas de información- Ambos poseen técnicamente las misma fases o etapas que definen a estos ciclos. Estas fases o etapas son:

  • Planificación: Consiste en el análisis de los costos y beneficios, programación estimación de recursos y asignación. Igualmente es el momento donde se recopilan los requisitos del sistema. (Amazon, S.F.)
  • Diseño: Se analizan los requisitos y se identifican las mejores soluciones para crear el software. (Amazon, S.F.)
  • Implementación: Donde el equipo de desarrollo codifica el producto. Se identifican los requisitos para dividirlos en pequeñas tareas que se puedan ir realizando diariamente hasta conseguir el resultado final. (Amazon, S.F.)
  • Pruebas: Se desarrollan pruebas automáticas y manuales para comprobar si el software tiene algún algún error. También se hacen los análisis de calidad para ver si el software cumple con los requisitos del cliente. Esta fase suele producirse en paralelo a la fase de desarrollo. (Amazon, S.F.)
  • Despliegue: Se lleva la última copia compilada al entorno de producción, configuración del entorno e instalación para finalmente desplegar el programa. (Amazon, S.F.)
  • Mantenimiento: Donde se corrigen errores, se resuelven problemas de los clientes y se administran los cambios hechos en el software.Aparte, se supervisa el rendimiento general del sistema, la seguridad y la experiencia del usuario para identificar nuevas formas de mejorar el software ya existente. (Amazon, S.F.)
